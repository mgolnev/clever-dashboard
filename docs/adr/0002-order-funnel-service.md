# 0002. Сервис воронки пути заказа

- Статус: Принято
- Дата: 2026-05-30

## Контекст

Нужно показать, где теряются заказы: гросс-заказ из-за проблем (неоплата онлайн,
отмена, невыкуп) не доходит до выполнения. Требуется воронка на отдельной вкладке
и разрезы по способу оплаты, доставке, каналу, региону.

## Решение

1. **Отдельный атомарный сервис `services/funnel`** — анализ пути заказа вынесен
   из `metrics`, т.к. это самостоятельная аналитическая ответственность
   (кумулятивные стадии, сегментные конверсии, проблемы). Не импортирует `metrics`
   и `orders`; работает с общей БД.
2. **Кумулятивная модель стадий**: created → paid → processing → shipped →
   delivered → completed. Заказ учитывается на стадии, если дошёл хотя бы до неё
   (на основе `status_stage` + `is_paid`).
3. **Разрезы** возвращаются по 4 справочникам сразу (payment/delivery/channel/
   region) с метриками оплата%/отмена%/выкуп% и числом проблем.
4. **Импорт обогащён полями** `has_problem`, `problem_desc`, `cancel_reason`
   (миграция через `CREATE` + идемпотентные `ALTER`), чтобы воронка показывала
   топ проблем сборки и причин отмены.
5. **ASP** (средняя цена позиции = выручка позиций / единицы) добавлен в KPI
   обзора рядом с AOV.

## Последствия

- Воронка и обзор независимы; изменение одной не ломает другую.
- Видны точки отвала и их зависимость от сегмента (напр. способ оплаты).
- Поля проблем/причин доступны для дальнейшей аналитики и алертов.
