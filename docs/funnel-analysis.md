# Анализ пути заказа и модель воронки

Документ описывает, как из данных Битрикса строится воронка и какие выводы она
помогает увидеть. Цифры ниже — по полной выгрузке (1328 заказов, сен 2025 – май
2026); в сервисе всё считается за выбранный период.

## Жизненный цикл заказа

Гросс-заказ (создан) → оплата онлайн → сборка/обработка → отправка → доставка в
ПВЗ → выкуп (Выполнен). На каждом шаге возможен отвал: неоплата → авто-закрытие,
отмена, проблема на сборке, невыкуп, возврат.

### Маппинг статусов Битрикса в стадии

| Статус | Трактовка |
|--------|-----------|
| Не подтвержден / Принят | создан, ждёт оплаты |
| Закрыт (неоплачен) ≈ Отменён | брошенная оплата / отмена |
| Оплачен | оплата прошла |
| Собирается / Обработан / Ожидает отправку | в сборке |
| Отправлен | отгружен |
| Прибыл в ПВЗ | доставлен, ждёт выкупа |
| Выполнен | выкуплен (успех) |
| Возврат заказа / частичный возврат | возврат после доставки |

Стадии воронки **кумулятивны**: заказ учитывается на стадии, если дошёл хотя бы
до неё (напр. «Выполнен» уже прошёл «Отправлен»).

## Ключевые выводы из данных

1. **Главный отвал — оплата онлайн.** Гросс 1328 → оплачено 996 (**75%**), т.е.
   ~25% (≈305 заказов) закрываются неоплаченными. Поле «Причина отмены» почти не
   заполняется (98% пусто) → это брошенная онлайн-оплата, а не явные причины.
2. **Способ оплаты — главный фактор отмены:**
   - Яндекс Пэй Сплит — оплата **57.7%**, отмена **41.8%** (хуже всех по объёму);
   - Яндекс Пэй — оплата 64.5%, отмена 35.5%;
   - Картой — оплата 75.9%, отмена 22.6%;
   - **СБП — самый здоровый: оплата 81.4%, отмена 18.6%**.
3. **Канал app/сайт не влияет** на оплачиваемость (~75% у обоих) — проблема не в
   платформе, а в платёжном сценарии.
4. **Доставка:** Яндекс ПВЗ лучший по оплате (83.6%), Почта России и самовывоз
   ~72%, СДЭК 68%.
5. **Сборка/отгрузка:** 33 проблемных заказа — нехватка остатка
   («Недостаточное количество товара …») и ошибки фискализации чека.
6. **Выкуп:** из отправленных до «Выполнен» доходит ~86% (часть свежих заказов
   ещё в пути — стадии после «Отправлен» частично отражают незавершённость).

## Что с этим делать (гипотезы)

- Разобрать сценарий оплаты Яндекс Сплит/Пэй (таймауты, ретраи, напоминания о
  неоплате) — это крупнейший управляемый рычаг выручки.
- Подсветить СБП как приоритетный способ оплаты.
- Контроль остатков перед публикацией (проблемы «недостаточное количество»).
- Мониторинг ошибок фискализации чеков.

## Разрезы воронки в сервисе

Вкладка «Воронки» строит кумулятивную воронку и разрезы по: **способу оплаты,
службе доставки, каналу, региону** — с метриками оплата% / отмена% / выкуп% и
подсветкой проблемных сегментов, плюс топ проблем сборки и причин отмены.
