# Импорт из Битрикса (MVP-1)

## Формат файла

Выгрузка «Заказы» из Битрикса. Поддерживаются:

- **HTML-«xls»** — фактический формат экспорта Битрикса (HTML-таблица с
  расширением `.xls`). Определяется автоматически по содержимому.
- **CSV** — UTF-8 или Windows-1251, разделитель `,` или `;` (автоопределение).

Гранулярность: **одна строка = один заказ**. Позиции товаров лежат в одной
ячейке столбца «Позиции».

## Используемые поля

| Поле выгрузки | Назначение |
|---------------|-----------|
| `Номер заказа` | Ключ дедупликации (PK) |
| `Дата создания` / `Дата изменения` | Время заказа (формат `ДД.ММ.ГГГГ ЧЧ:ММ:СС`) |
| `Сумма` | Сумма заказа (`«3 313 руб»` → `3313`) |
| `Отменен` / `Оплачен` | Флаги (Да/Нет) |
| `Статус` | Сырой статус → стадия воронки |
| `Платежная система` / `Служба доставки` | Срезы оплаты/доставки |
| `Заказ из приложения` | Канал: «Приложение» / «Сайт» |
| `Выберите свой населенный пункт` | Регион/город (`Россия, край, город`) |
| `Позиции` | Товарные позиции: `[offer_id] Название (N шт)` |
| `Цена товара` | Цены позиций (`«719 руб 503 руб»`), сопоставляются по порядку |

## Разбор позиций

Столбец «Позиции» парсится регулярным выражением
`\[(\d+)\]\s*(.*?)\s*\((\d+)\s*шт\)`. Цены берутся из столбца «Цена товара»
(разделитель «руб») и сопоставляются по индексу — соответствие 1:1 проверено на
полной выгрузке (1328 заказов → 4823 позиции, 0 расхождений).

Из названия товара извлекаются атрибуты:

- `brand` — 1-е слово (CLEVER, CLES, ЭЙС…);
- `category` — 2-е слово (Футболка, Брюки, Платье…);
- `gender` — 3-е слово, нормализуется (Мужской/Женский/Детский/Унисекс);
- `size` — последний короткий токен.

## Маппинг статусов в воронку

| Статус Битрикса | Стадия |
|-----------------|--------|
| Не подтвержден | `new` |
| Принят / Обработан / Ожидает отправку / Собирается | `processing` |
| Отправлен | `shipped` |
| Прибыл в ПВЗ | `in_pvz` |
| Выполнен | `completed` |
| Закрыт | `closed` |
| Возврат заказа / Частичный возврат | `returned` |
| (флаг «Отменен» = Да) | `canceled` (приоритет) |

## Идемпотентность

Повторная загрузка обновляет заказы по `Номер заказа` (UPSERT), позиции
пересоздаются. Дублей не возникает — можно безопасно загружать пересекающиеся
периоды.
