## Когда заводить ADR

- новый слой, контракт API, схема БД;
- труднообратимое решение (движок поиска, миграция, источник данных);
- согласование backend / frontend.

Без ADR: косметика, точечные баг-фиксы, узкие фичи без архитектурного следствия.
