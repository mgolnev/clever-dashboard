# Словарь полей выгрузки Битрикса (заказы)

Справочник по выгрузке «Заказы» CleverWear (`sale_order.xls`, формат HTML-таблицы).
**117 колонок, 1 строка = 1 заказ.** Колонка «Заполнено» — доля непустых значений
в эталонной выгрузке (1328 заказов); помогает понять, что реально пригодно для
анализа. «В сервисе» — поле перенесено в витрину (`orders`/`order_items`).

> При анализе данных по заказам сверяйтесь с этим файлом: он фиксирует смысл
> полей, их типы и связи. Аналитическая модель воронки — в
> [funnel-analysis.md](funnel-analysis.md), контракт импорта — в
> [import-bitrix.md](import-bitrix.md).

## Главные связи (резюме)

- `Номер заказа` → 1:N `order_items` (через разбор `Позиции`).
- Товарные столбцы (`Позиции`/`Цена товара`/`Количество товара`/`Код товара`/…)
  — **параллельные массивы**, элементы соответствуют друг другу по индексу.
- `Статус` + `Оплачен` + `Отменен` задают стадию пути заказа: неоплачен →
  «Закрыт»/«Отменён»; оплачен → сборка → отправка → ПВЗ → «Выполнен»; возвраты —
  отдельная ветка.
- Адресные поля выбираются по `Служба доставки` (для каждой службы — свой набор).
- `Сумма` = товары + `Стоимость доставки`; `Оплачено`/`Сумма ПС` — факт оплаты.
