# Архитектура

## Принципы

- **Атомарные независимые сервисы** (см. AGENTS.md): один модуль — одна
  ответственность; домены не импортируют внутренности друг друга.
- **Коннекторное ядро**: источники данных (файл Битрикса, в будущем Bitrix API,
  Метрика, Аппметрика) подключаются как сменные входы, не затрагивая витрину.
- **Переносимость БД**: один код работает на SQLite и Postgres (`db.IsPostgres()`).

## Поток данных

```
Файл XLS/CSV ──► ingestion ──► normalize ──► orders (raw_import, orders, order_items)
                                                  │
                                                  ▼
                                              metrics ──► HTTP API ──► React-дашборд
```

## Модули (Go)

| Пакет | Ответственность | Зависит от |
|-------|-----------------|------------|
| `ingestion` | Парсинг файла Битрикса (HTML/CSV), разбор позиций | `normalize`, `model` |
| `normalize` | Деньги, гео, статусы→стадии, атрибуты товара | — |
| `services/orders` | Импорт, дедуп, хранение витрины | `ingestion`, `model`, `db` |
| `services/metrics` | KPI, воронка, срезы, сравнение периодов | `db` |
| `handlers` | HTTP (Fiber), валидация входа | `container` |
| `container` | DI: сборка зависимостей | все сервисы |

`orders` и `metrics` не вызывают друг друга — связь только через общую БД и
нейтральный пакет `model`.

## Модель данных

- **`raw_import`** — журнал загрузок (файл, период, счётчики). История и аудит.
- **`orders`** — нормализованный заказ (PK = `order_number`, дедуп при повторной загрузке).
- **`order_items`** — позиции заказа (из столбца «Позиции») с атрибутами товара.

Время хранится строкой `YYYY-MM-DD HH:MM:SS` — лексикографически сравнимо в
SQLite и корректно приводится в Postgres; фильтры периода работают одинаково.

## Расчёт периодов

Длина периода `L = end − start + 1` дней. Предыдущий период —
`[start − L, start − 1]` (та же длина, примыкает слева). Сравнение метрик
ведётся именно с ним (например, неделя Пн–Вс сравнивается с прошлой Пн–Вс).

## Дальнейшее развитие

- Bitrix REST API как альтернативный `ingestion`-вход (этап MVP-2).
- Коннекторы `connectors/metrika`, `connectors/appmetrica` (этапы 3–4) дополняют
  `metrics` верхом воронки (трафик, визиты, конверсия) без изменения витрины заказов.
